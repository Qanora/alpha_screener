# LLM 重定位为风险审计层

AlphaScreener 的 LLM（GPT-4o-mini）相对 13 因子量化模型没有信息优势。因此 LLM 不担任最终概率预测器，而是定位为**风险审计层**：输出 score_correction（[0.90, 1.05] 的有限修正乘数）+ risk_tags（结构化风险标签），对粗筛 Coarse_Score 做边际调整。LLM 的真正价值在于识别因子模型无法捕捉的结构化风险（数据源冲突、催化剂时序错位、流动性陷阱、退市风险），而非替代因子模型的预测功能。

Ablation 双轨强制验证：每日并行记录纯因子轨和含 LLM 轨的 Lift@20，60 天 ΔLift@20 ≥ 0.05 才允许 LLM 参与最终排序。不达标则 LLM 仅作可解释性输出。

**Considered Options**: LLM 端到端预测器（放弃 — LLM 对 T+7 股价无信息优势，强制概率预测会失去区分度）、多模型投票（放弃 — V1.5 前不做，增加成本无边际收益）。
